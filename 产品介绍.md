## 一、产品基础定位

**产品背景**：B端产品经理面试竞争激烈，候选人普遍面临三大痛点：面试准备缺乏针对性、模拟面试缺少真实反馈、面试过程无系统沉淀。市面现有面试工具偏通用型，缺少垂直 B 端 PM 领域的深度能力。

**使用人群**：B端产品经理求职者（初中级 → 资深均可），以「素材管理 → 面试准备 → 模拟演练」为核心使用路径。

**核心解决痛点**：
1. 简历/JD/项目素材散落多处，缺少统一解析与结构化画像
2. 面试准备靠零散搜索，缺少基于个人素材的定制化学习材料
3. 缺少真实面试场景的角色扮演练习和量化评估反馈

**产品价值**：一站式 B 端 PM 面试全流程 Agent，覆盖「素材入库 → 画像生成 → 定制准备 → 模拟面试 → 评估反馈」五阶段闭环，深度打通 Obsidian 个人知识库。

## 二、完整功能清单

### 2.1 素材库管理
| 功能 | 说明 |
|------|------|
| 文件导入 | 支持上传简历、项目文档、JD（.txt/.md/.pdf/.xlsx/.xls），图片素材（.png/.jpg/.webp），LLM 自动提取结构化信息（Excel 转 markdown 表格后提取），保留原始文件名 |
| 分类管理 | 自动归类为简历/项目/JD/图片四大类别，维护集中索引 |
| 全文搜索 | 关键词精确匹配 + LLM 语义搜索双模式，同步搜索 Obsidian Vault |
| 内容查看 | 素材列表中可直接点击查看文件全文内容 |
| 素材删除 | 支持单文件删除，同步清理索引和物理文件 |
| 候选人画像 | 基于全部素材综合生成结构化画像（技能/经历/亮点/短板） |
| Obsidian 集成 | 已深度打通：素材搜索同步搜索 Vault、面试准备自动引用 Vault 内容、Web UI 独立 Obsidian 面板 |

### 2.2 面试准备
| 功能 | 说明 |
|------|------|
| 目标岗位设定 | 输入目标公司+岗位，系统基于素材和 JD 生成定制学习材料 |
| 岗位分析 | LLM 分析 JD 核心要求，输出能力权重判断 |
| 匹配度分析 | 候选人素材 vs JD 要求对比，定位优势和短板 |
| 面试题预测 | 基于 JD 要求预测高频面试题（含出题逻辑说明） |
| 对话问答 | 基于候选人真实素材的多轮问答，引导答题而非代答 |
| 网络搜索 | DuckDuckGo 联网补充行业/公司信息 |

### 2.3 模拟面试
| 功能 | 说明 |
|------|------|
| 角色扮演 | LLM 扮演指定公司的 B 端产品面试官，模拟真实面试对话 |
| 简历库/岗位库 | 支持从素材库下拉选择简历和 JD，也支持直接上传本地文件，面试官基于选定资料面试 |
| 多题型覆盖 | 行为面试/产品 Sense/估算题/策略题/执行题自动穿插 |
| 追问机制 | 面试官根据回答质量自动深挖 1-2 轮 |
| 答题提示 | 候选人可请求提示（给框架不给答案） |
| 量化评估 | 5 维度评分：逻辑架构/B 端思维/数据意识/岗位匹配度/表达质量 |
| 评估报告 | 优势/不足/优先级改进建议/示范回答 |

### 2.4 已移除
面试追踪模块（tracker）已在 v1.1 移除，项目现聚焦于三大核心模块：素材库、面试准备、模拟面试。

## 四、大模型选型说明

| 项目 | 选择 |
|------|------|
| **主模型** | DeepSeek-Chat（via DeepSeek API） |
| **温度** | 0.3（结构化提取/评估）~ 0.7（面试官角色扮演） |
| **上下文窗口** | 默认 4K tokens，素材输入截断至 15K 字符 |
| **JSON Mode** | 开启（画像生成、素材提取、面试评估） |

**选型理由**：DeepSeek API 成本低（约为 GPT-4 的 1/20）、中文能力强、支持 JSON Mode。作为个人学习项目，优先成本控制。LLM 层通过 provider 抽象设计，支持一键切换到 Anthropic Claude 或 OpenAI。

## 五、Skill 工具能力

### 5.1 material — 素材库管理
| 命令 | 触发条件 | 功能 | 输出 |
|------|---------|------|------|
| `material import <路径>` | 用户导入文件 | 复制文件 + LLM 提取结构化信息 + 更新索引 | 结构化预览 |
| `material list` | 查看素材 | 读取本地索引和文件系统 | 树状目录 |
| `material search <关键词>` | 搜索素材 | 精确匹配 → 失败则 LLM 语义搜索 | 搜索结果表 |
| `material profile` | 生成画像 | LLM 综合分析全部素材 | 结构化画像卡片 |

### 5.2 prep — 面试准备
| 命令 | 触发条件 | 功能 | 输出 |
|------|---------|------|------|
| `prep for <公司> <岗位>` | 设定目标 | LLM 基于素材生成定制材料 | 岗位分析+匹配度+题预测+建议 |
| `prep ask <问题>` | 面试问答 | 基于素材引导答题 | 问题分析+框架+素材参考 |
| `prep search <关键词>` | 联网搜索 | DuckDuckGo 搜索 | 搜索结果摘要 |
| `prep save-note` | 保存笔记 | 保存最近一轮问答 | 确认 → 入库 |

### 5.3 mock — 模拟面试
| 命令 | 触发条件 | 功能 |
|------|---------|------|
| `mock start <公司> <岗位>` | 开始面试 | 创建面试会话，LLM 扮演面试官出题 |
| `mock answer <回答>` | 回答问题 | 保存答案，面试官追问或出下一题 |
| `mock hint` | 请求提示 | 给答题框架和切入角度（不给答案） |
| `mock end` | 结束面试 | 保存会话，计算状态 |
| `mock review` | 查看评估 | LLM 5 维度评估 + 优势/不足/建议/示范回答 |

### 5.4 obsidian — Obsidian 集成
| 命令 | 功能 |
|------|------|
| `obsidian scan` | 重建 Vault 文件索引 |
| `obsidian tree` | 查看 Vault 目录树结构 |
| `obsidian search <关键词>` | 全文搜索 Vault 内容 |
| `obsidian read <序号>` | 预览搜索结果中的文件 |
| `obsidian import <序号>` | 将搜索结果文件导入素材库 |

## 七、完整工作流

### 7.1 素材库导入链路
```
用户上传文件 → 识别文件类型（简历/JD/项目/图片）
  → 图片：直接复制到 data/materials/raw/images/，记录元数据
  → Excel：openpyxl 读取转 markdown 表格 → LLM 提取结构化信息
  → 文本/PDF：read_text/pdfminer 提取 → LLM 提取结构化信息
  → 更新 data/materials/index.json 索引
  → 如需生成画像：汇总全部文本素材 → LLM 综合分析 → 保存 profile.json
```

### 7.2 面试准备链路
```
用户设定目标 → 加载候选人画像 + 素材摘要
  → LLM 分析 JD 核心要求 + 匹配度 + 高频题预测
  → 用户提问 → 构建上下文（画像+素材）
  → LLM 回答（prep_assistant prompt：引导框架而非代答）
  → 可选：搜索互联网补充信息
```

### 7.3 模拟面试链路
```
用户选择简历（素材库/本地上传）+ JD（素材库/本地上传）
  → 加载简历内容 + JD 内容（支持 txt/md/pdf/xlsx）
  → LLM 初始化面试官角色（interviewer prompt）
  → 面试官自我介绍+出第一题
  → 用户回答 → LLM 判断追问还是出下一题
  → 循环至 10 题或手动结束
  → LLM 5 维度评估（evaluator prompt）→ 输出报告
```

### 7.4 Obsidian 集成链路
```
配置 vault_path → 首次扫描建索引（缓存到本地）
  → material search 同步搜索 Vault + 素材库
  → prep ask 自动注入 Vault 概览到上下文
  → Web UI Obsidian 面板：搜索 + 预览 + 一键导入
```

## 八、提示词工程

| Prompt 文件 | 角色设定 | 关键约束 | 输出格式 |
|------------|---------|---------|---------|
| `material_extract.txt` | B 端 PM 面试素材分析师 | 保留量化数字、按文件类型提取不同字段 | JSON |
| `prep_assistant.txt` | B 端 PM 面试准备助手 | 基于真实素材、引导而非代答、指出可用项目经历 | 结构化文本 |
| `interviewer.txt` | 指定公司 B 端 PM 面试官 | 5 种题型穿插、追问 1-2 轮、保持专业不评价 | 面试官发言 |
| `evaluator.txt` | 资深 B 端 PM 面试教练 | 5 维度评分、具体反馈、示范回答 | JSON |

## 九、异常与边界处理

- **LLM 调用失败**：抛出 RuntimeError 并显示错误详情，不静默吞异常
- **文件不存在**：返回明确红色错误提示，提示用户检查路径
- **JSON 解析失败**：清理 markdown 包围后重试，仍失败则抛异常
- **素材库为空**：返回引导提示「请先导入素材」
- **未生成画像**：检查素材是否为空，引导用户先执行 material profile
- **模拟面试最大题数**：硬限制 10 题，超限自动结束并提示查看评估
- **配置缺失**：LLM provider/api_key 支持环境变量 → config.yaml 二级回退
- **token 超限**：素材文本截断至 15K-30K 字符后送入 LLM
